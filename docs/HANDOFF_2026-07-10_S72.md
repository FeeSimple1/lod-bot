1. Rules first, cite everything. New reference conflicts go to `QUESTIONS.md` and STOP that item; keep working elsewhere.

3. Playbook goldens are the deep instrument — flowchart-level sweeps do NOT retire body-level risk.
4. Environment gotchas (hard-won):
   * Work from a clone in the sandbox home (`~/work`), NOT a Windows-mounted folder. The Read/Edit file tools point at a DIFFERENT filesystem than the sandbox — edit the repo via bash heredocs only.
   * Edit via literal-file replacement: `cat > /tmp/old.txt <<'EOF'` … then a Python driver does `assert src.count(old)==1; src.replace(old,new)`. Single-quoted heredocs pass content literally (brackets/quotes safe). Always re-grep / syntax-check after a failed script.
   * `pip install pytest mypy --break-system-packages` in a fresh sandbox; run pytest with `-p no:cacheprovider`.
   * Delete the old `soak.jsonl` before a fresh soak (it resumes completed files → instant-DONE).
   * Gate/balance/soak need `PYTHONHASHSEED=0`. Keep bash calls <= ~40s; soak in ~38s slices and re-run until DONE. Full suite ~55s — split it.
   * Non-player SPACE ties: `random_spaces.pick_by_priority(state, [(key, sid), …])` with SUBSTANTIVE keys only (never rng in a sort key, Q22). FACTION/non-space ties: explicit seeded roll or `util/target_order` — the table is space-only.
   * `normalize_state` runs on the hot path (every commit + sandbox); new invariants there must be deterministic and rng-free.
   * The seeded-rng lint (`tests/test_no_global_random.py`) will fail CI if gameplay code touches the global `random` module — use `state["rng"]`.
   * Subagent audits work for read-only sweeps BUT verify every finding yourself — false positives are real.

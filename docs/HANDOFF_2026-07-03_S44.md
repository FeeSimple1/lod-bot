## Working method (established — follow it)

1. Read the full rule section before touching code. Reference
   conflicts go to QUESTIONS.md and STOP; rulings are case-by-case
   (Q16 manual-over-flowchart, Q17 specific-flowchart, Q18
   specific-bullet-over-general-3.1 — no precedence rule exists).
2. Battery per behavioral change: both test roots (chunk
   `lod_ai/tests` into thirds — a full root exceeds the 45s bash cap),
   gate `--seeds 1-10` then `11-20`, soak via repeated
   `python -m lod_ai.tools.soak --games 120 --invariants
   --max-seconds 36` until DONE, `balance_smoke` per scenario
   (`--scenarios 1775` etc. — all three at once exceeds 45s), then
   `--update` per scenario if winners flipped, with an old-vs-new
   distribution sanity check. Tests encoding superseded behavior get
   REWRITTEN to the rule with citations.
3. **Environment gotchas (hard-won this stretch):**
   - Work from a clone in the sandbox home (`~/work`), NOT a
     Windows-mounted folder. The file-tool Edit/Write path silently
     TRUNCATES large files on mounted folders (four corruptions in
     Session 38), and a stale `.git/index.lock` on a mount cannot be
     removed. Make ALL edits via `python3 - <<'PYEOF'` string
     replacement with `assert src.count(old) == 1`, and re-verify
     imports after each edit.
   - Background processes do NOT survive between bash calls, and
     `pgrep` false-positives on the wrapper's own command line.
   - Gate/balance replays need `PYTHONHASHSEED=0`.
   - `french.py` contains U+202F characters; `_bot_error_log` entries
     key `exception_message` (the gate printer shows None).
4. Balance samples are 20 games/scenario: the 1778 British rate has
   see-sawed 15→5→20→5 across Sessions 38–44 on comparable rng-stream
   shifts. Don't chase per-session strength readings; Piece 8
   (large-N stats) is the instrument.

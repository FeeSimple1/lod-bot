## Working method (established, follow it)

1. **Read the full rule section before touching code.** Three
   documented incidents this stretch came from partial readings
   (T2's false "no icon data" claim; the missing sheet "found" at the
   bottom of the flowchart files, keyed by card TITLE not number; the
   friendly/enemy 8.1.2 bullet mix-up).
2. Reference conflicts go to QUESTIONS.md and STOP (agents.md). Eric's
   rulings are case-by-case: Q16 went manual-over-flowchart, Q17 went
   specific-flowchart-over-general-manual. Do NOT generalize either
   into a precedence rule; escalate each new conflict.
3. Battery per behavioral change: both test roots (run each root
   separately — the sandbox bash has a 45s cap), clean-sweep gate as
   `--seeds 1-10` then `11-20`, soak via repeated
   `python -m lod_ai.tools.soak --games 120 --invariants
   --max-seconds 36` until DONE (background processes do NOT survive
   between bash calls), then `balance_smoke --update` if pinned
   winners flipped, with an old-vs-new distribution sanity check
   (`git show HEAD:lod_ai/tools/balance_baseline.json`).
4. Tests encoding superseded behavior get REWRITTEN to the rule, with
   the rule cited in the docstring.
5. Gotchas: `french.py` strings contain U+202F (narrow no-break
   space) — exact-match patches can silently miss; `_bot_error_log`
   entries key `exception_message` (the gate's detail printer shows
   None); replay a dirty gate game with the snippet in audit_report
   Session 23 to extract tracebacks.
